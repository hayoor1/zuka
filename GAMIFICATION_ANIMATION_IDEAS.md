## üéØ **GENERAL ANIMATION PRINCIPLES**

### **Performance**
- Use CSS transforms (translate, scale, rotate) instead of position/width changes
- Use `will-change` property sparingly
- Prefer GPU-accelerated properties
- Limit simultaneous animations
- Use `requestAnimationFrame` for complex animations

### **Timing**
- **Micro-interactions**: 150-300ms (hover, click)
- **Transitions**: 300-500ms (card movements)
- **Celebrations**: 1-2 seconds (level up, achievements)
- **Background**: 3-10 seconds (ambient effects)

### **Easing**
- **Ease-out**: For entrances (feels natural)
- **Ease-in-out**: For transitions (smooth)
- **Bounce**: For celebrations (playful)
- **Elastic**: For special moments (delightful)

### **Accessibility**
- Respect `prefers-reduced-motion` media query
- Provide option to disable animations
- Ensure animations don't cause motion sickness
- Keep essential info visible during animations

## üöÄ **TECHNOLOGY RECOMMENDATIONS**

### **For Simple Animations**
- **CSS Animations**: Best for simple, declarative animations
- **CSS Transitions**: For hover states and micro-interactions
- **Tailwind CSS**: Use built-in animation utilities

### **For Complex Animations**
- **Framer Motion**: React animation library (smooth, performant)
- **GSAP**: Professional animation library (powerful, flexible)
- **Lottie**: For complex vector animations (After Effects exports)

### **For Particles**
